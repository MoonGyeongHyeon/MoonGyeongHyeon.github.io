적용 기술


Retrofit + GSON

HTTP REST API 통신 라이브러리와 JAVA Object - JSON 파싱 라이브러리를 통해 박물관 관련 데이터를 받아왔습니다.

Glide

대표적인 이미지 로딩 및 캐싱 처리 라이브러리인 Glide와 Picasso 중 속도에 우선순위를 둔 Gilde를 사용하여 박물관 이미지를 보여주도록 했습니다.



Parceler

Class를 손쉽게 Parcelable 인터페이스를 확장한 객체로 만들어주는 라이브러리를 적용하여, 박물관 객체나 이야기 객체를 인텐트로 전달했습니다.

ButterKnife

안드로이드 소스 바인딩 라이브러리를 이용하여 보일러 플레이트를 줄이고 소스 관리를 보다 깔끔하게 했습니다.



Toasty

Custom Toast 생성을 손쉽게 지원해주는 라이브러리를 적용하여 서버 연결 실패 안내, 예외 처리에 대한 알림 등을 보기 좋게 출력했습니다.

Fresco ImageViewer

Fresco Library를 전체 화면으로 간단하게 커스터마이징할 수 있도록 돕는 라이브러리를 적용하여 박물관 이미지 크게보기나 박물관 이야기를 보기 좋게 꾸미는 데 사용했습니다.



node.js + Express + Mysql

서버 사이드 플랫폼인 node.js와 웹 프레임워크인 Express, RDBMS 중 하나인 Mysql을 이용하여 데이터베이스 서버를 구축했습니다.
